# Proposed Folder Structure Strategy

## Root Config
- .vercelignore
- vercel.json
- package.json
- README.md

## Assets
- /assets/
  - /images/
  - /data/
  - /spline/
  - favicon.ico
  - robot.txt
  - schema.json

## CSS System
- /css/
  - design-system.css    (Core variables & components)
  - style.css            (Page-specific overrides if absolutely needed, try to avoid)

## JavaScript System
- /js/
  - app.js               (Main logic: Nav, Lazy loading)
  - spline-loader.js     (Optimized 3D loader)
  - forms.js             (Contact form handling)

## Pages
- index.html             (Home)
- /pages/
  - pricing.html
  - services.html
  - about.html
  - contact.html
  - /case-studies/
       - index.html
       - [case-study-slug].html
  - /legal/
       - privacy.html
       - terms.html

## Build Artifacts
- /dist/                 (Generated by build script)

---

## Migration Plan

1.  **Replace CSS**: Add `design-system.css` to all pages. Remove `styles.css`, `hyperspeed.css`, `interactive-button.css`.
2.  **Standardize HTML**: Copy the Navbar and Footer HTML blocks from `examples.html` to every page.
3.  **Refactor Hero**: Replace the entire hero section on `index.html` with the new structure in `examples.html`.
4.  **Refactor Pricing**: Replace pricing cards on `pages/pricing.html` with the new `.card-price` structure.
5.  **Cleanup**: Delete unused CSS/JS files.
